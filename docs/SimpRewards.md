## 一、奖励规则体系（双轨制）
### 轨道一：每日上线奖励（签到）
+ **重置时间**：每天 00:00（现实时间，服务器时区）
+ **领取条件**：玩家在当天 00:00 之后首次登录，或登录后手动点击签到按钮
+ **奖励内容**（全部由配置文件自定义）：
    - **基础奖励**：可配置任意物品、经济、经验、命令等
    - **连签加成**：可配置不同连签天数对应的不同奖励（例如第 1~3 天一套奖励，第 4~6 天另一套，第 7 天特殊奖励）
    - 连签周期可配置（如 7 天重置或无限累加）
+ **补签机制**：不补签，断签则连击归零（可配置是否允许补签）

---

### 轨道二：在线时长奖励（累计 / 每日 / 连续）
#### 1. 总累计在线时长奖励（终生唯一）
+ 按玩家累计在线总时长达到指定阈值时，可领取 **一次**
+ **阈值与奖励**：完全由配置文件定义，支持任意数量的阶梯（例如 30分钟、1小时、2小时…… 每个阶梯独立配置奖励）
+ **领取方式**：手动打开 GUI 点击领取

#### 2. 每日在线任务（当日有效，每天重置）
+ **目标**：当日累计在线满 **X 分钟**（X 可配置，例如 30）
+ **奖励**：由配置文件定义（可包含物品、经济、经验、命令等）
+ **重置**：每天 00:00 重置任务进度和领取状态
+ **领取**：满足时长后，GUI 中按钮由灰变亮，点击领取

#### 3. 每周在线任务（周常）
+ **目标**：本周累计在线满 **? 分钟**（配置文件决定）
+ **重置**：每周一 00:00
+ **奖励**：配置文件定义
+ **领取**：手动领取，每周只能领一次

#### 4. 连续在线天数奖励（日历连续）
+ **规则**：每日至少在线 **Z 分钟**（Z 可配置）才计为“活跃一日”
+ **连续天数档次**：完全由配置文件定义（例如连续 3 天、7 天、14 天、30 天……），每个档次独立配置奖励
+ **中断处理**：某一天未达到活跃门槛 → 连续天数归零
+ **领取**：达到对应天数时，GUI 中出现可领取的按钮（每个档次仅能领一次）

> 所有上述奖励的具体内容（物品类型、数量、经济金额、经验值、执行的命令、称号等）均在配置文件中自由设置，插件不内置任何固定奖励示例。
>

## 二、界面设计（GUI）
### 主界面入口
+ 命令：`/reward` 或 `/daily`（可配置）
+ 可通过菜单插件绑定自定义物品打开

### GUI 布局（建议 4×9 = 36 格）
#### 第 1 行（奖励总览）
| 槽位 | 图标 | 名称 | 说明 |
| --- | --- | --- | --- |
| 0 | 时钟 | **每日签到** | 显示今日是否已签 / 连签天数，可点击领取 |
| 1 | 钻石剑 | **累计在线** | 显示总累计时长 & 下一档奖励，可点击领取 |
| 2 | 绿宝石 | **每日任务** | 显示今日已在线分钟 / 目标值，可点击领取 |
| 3 | 下界合金锭 | **每周任务** | 显示本周已在线小时 / 目标值，可点击领取 |
| 4 | 附魔书 | **连续在线** | 显示当前连续活跃天数，可点击领取对应档次 |
| 5~8 | 装饰玻璃 | (留空或展示服务器信息) | - |


#### 第 2 行（具体状态及奖励槽）
+ **槽位 9**：签到状态图标  
    - 未签到：可用金锭（可配置） → Lore 显示连签天数  
    - 已签到：可用绿宝石块（可配置） → Lore 显示今日已签
+ **槽位 10**：累计时长奖励进度条  
    - 物品：时钟（可配置）  
    - Lore 显示：`总在线：xxx 分钟` `下一档：xxx 分钟 奖励：xxx`（奖励描述从配置文件读取）  
    - 若已达阈值未领 → 图标变为烈焰棒（可配置），Lore 提示“点击领取”
+ **槽位 11**：每日任务进度  
    - 物品：日晷（可配置）  
    - Lore：`今日在线：X / Y 分钟`  
    - 完成后图标变亮（可配置物品），可点击领取
+ **槽位 12**：每周任务进度  
    - 物品：周历（可配置）  
    - Lore：`本周在线：X / Y 小时`  
    - 完成后图标变亮（可配置物品）
+ **槽位 13**：连续在线奖励面板  
    - 物品：钻石（可配置）  
    - Lore：`连续活跃：N 天` `下一奖励：M 天 → 奖励描述`  
    - 若已达到对应天数未领 → 图标闪烁或替换（可配置），可点击领取
+ **槽位 14~17**：装饰 / 服务器公告（可配置）

#### 第 3 行（额外展示 / 提示）
+ 放置书与笔：显示帮助文本（可配置）
+ 放置木牌：显示下次每日重置倒计时（可配置）
+ 放置屏障：关闭 GUI（可配置）

#### 第 4 行（底部装饰）
+ 全部用黑色染色玻璃板填充（可配置材质），避免误点

### 动态提示与反馈
+ **鼠标悬停**：Lore 中显示具体进度、所需条件和奖励预览（从配置文件读取）
+ **点击奖励槽**：
    - 满足条件 → 发放奖励（聊天栏提示 + 获取物品/经济/经验等），槽位变为已领取状态
    - 不满足条件 → 聊天栏或 ActionBar 提示缺少的条件（如“还需要在线 5 分钟”）
+ **已领取的奖励槽**：替换为灰色染料或障碍物（可配置），标注“已领取”

### 可配置界面元素
+ 所有 GUI 中的物品图标、名称、Lore 格式、提示文本、音效、标题文字等均可通过配置文件修改，无需改动代码。

## 三、规则总结表
| 奖励类型 | 重置周期 | 领取方式 | 奖励来源 | 备注 |
| --- | --- | --- | --- | --- |
| 每日签到 | 每天 00:00 | GUI 手动点击 | 配置文件定义（基础+连签） | 连签周期可配置 |
| 总累计时长 | 终身 | GUI 手动 | 配置文件定义（阶梯奖励） | 每档只一次 |
| 每日任务 | 每天 00:00 | GUI 手动 | 配置文件定义 | 需在线满 X 分钟（可配置） |
| 每周任务 | 每周一 00:00（可配置） | GUI 手动 | 配置文件定义 | 需在线满 Y 分钟（可配置） |
| 连续在线天数 | 日历连续 | GUI 手动 | 配置文件定义（档次奖励） | 每日需活跃 ≥Z 分钟（可配置） |
